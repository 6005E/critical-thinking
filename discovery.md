# Discovery

Load at task initiation when beginning original development, analysis, or design; establishing a new process; or working in any domain where prior work plausibly exists.

This file loads independently — its trigger is task initiation, not a mid-execution cognitive state.

---

Prior art and existing implementations are inputs to the design process, not just guardrails against duplication. Treat discovery as generative.

- At the initiation of any significant project, search for prior art, existing implementations, and related work before proceeding.
- Search for precedent proactively at task initiation — not only reactively when asked.
- Before producing original content, ask whether existing work would materially change the approach — and if so, find it first.
- When establishing a process, verify the process itself is complete before executing within it.
